<b>═ 🔥 TOP STORY ═</b>
CVE-2026-31431 "Copy Fail" is a Linux kernel local privilege escalation via the algif_aead module (AF_ALG cryptographic socket interface), carrying a CVSSv3 score of 7.8 and affecting every kernel ≥ 4.14 shipped since 2017. The flaw allows any unprivileged local user to modify the kernel's page-cache copy of a privileged binary without touching the on-disk file, then execute it to gain root. A working 732-byte Python PoC is publicly available and reported to work reliably across Debian, Ubuntu, RHEL, and Alpine without modification — CISA has added it to the Known Exploited Vulnerabilities catalog. For embedded engineers the threat surface is broad: any Linux-based IoT gateway, edge server, or dev machine reachable via SSH, web-shell pivot, or container escape is at full-root risk until patched. Fix: upgrade kernel to ≥ 6.18.22, ≥ 6.19.12, or ≥ 7.0; interim mitigation is to blacklist the algif_aead initcall via grubby to deregister the AEAD AF_ALG interface at boot.
