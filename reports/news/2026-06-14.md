<b>4. Post: Physics-meets-architecture — BLE 6.2 cuts the connection interval to 375 µs. Here is what that costs.</b>
Bluetooth LE just got a 20× faster connection interval. The spec does not tell you why you probably will not hit it.

Bluetooth Core 6.2 (November 2025) drops the minimum LE connection interval from 7.5 ms to 375 µs with 125 µs resolution. Sub-millisecond BLE data exchange — previously only achievable with wired buses or proprietary 2.4 GHz protocols — is now in the specification.

In practice: the BT stack needs to process the packet, run the connection-event state machine, handle application callbacks, and schedule the next event — all within 375 µs. Most host BT stacks cannot do this without carefully co-designing the application with the stack interrupt model. NovelbBits' hands-on with the Nordic nRF54L15 is the clearest public documentation of what you actually need to optimise to approach the lower bound.

The use cases where the physics matters most: closed-loop BLE sensor control, real-time HID, low-latency audio feedback. The constraint is not the spec — it is the round-trip through your stack.
