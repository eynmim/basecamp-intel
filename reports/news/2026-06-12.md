<b>3. Post: [Security wake-up] — Your vulnerability scanner was the attack vector</b>
Aqua Security's Trivy scanner — one of the most widely deployed open-source firmware and container vulnerability scanners in CI/CD pipelines — was itself compromised in March 2026 (CVE-2026-33634). A malicious version was published that harvested credentials from build environments. CISA added it to the Known Exploited Vulnerabilities catalog.

The embedded security angle that is easy to miss: firmware engineering CI pipelines increasingly run Binwalk, Trivy, or Grype for automated SBOM and vulnerability scanning. These tools run with the same privileges as the build system — often elevated. A compromised scanner silently exfiltrates signing keys, environment secrets, and private credentials that exist in the build environment.

The fix is not switching tools. It is treating your security tooling installation the same way you treat your compiler toolchain: pin binaries by cryptographic hash, verify signatures before execution, and make scanner version changes an explicit, reviewed decision.

An unsigned, unpinned security scanning binary in your CI pipeline is the same threat model as unsigned firmware in your device.
