<b>═ 🔥 TOP STORY ═</b>
CVE-2026-31431 "Copy Fail" — disclosed April 30 by Theori — is a logic flaw in the Linux kernel's algif_aead module (AF_ALG Crypto API) that lets any unprivileged local user gain root through deterministic, race-condition-free controlled 4-byte writes into the page cache backing executable files. Unlike Dirty Cow or Dirty Pipe, the exploit is reliable and reproducible across the entire kernel range from 2017 (kernel 4.9) through current releases, meaning every embedded Linux fleet — Yocto, Buildroot, Android, Ubuntu Core — running any kernel built in the last nine years is affected. CISA added it to the Known Exploited Vulnerabilities catalog and mandated US federal agency patches by May 15; major distributor patches are now available for kernel 6.1 LTS and 6.6 LTS. The stealth characteristic is especially dangerous for embedded deployments: the kernel does not flush the modified page cache to disk, so standard file-integrity monitoring, auditd, and post-compromise disk-image forensics all see the original unmodified binary — behavioural detection via privilege-escalation journal events is the only reliable on-device indicator. Immediate mitigation without a kernel bump: blacklist algif_aead via modprobe config.
