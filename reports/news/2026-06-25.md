<b>═ 🔥 TOP STORY ═</b>

CVE-2026-55200 dropped June 24: a CVSS 9.2 out-of-bounds heap write in libssh2's <i>ssh2_transport_read()</i> that fires before authentication — during the initial transport negotiation, the moment a client opens a connection. The root cause is a missing upper-bound check on the <i>packet_length</i> field; an attacker supplies an oversized value, triggering an integer overflow that writes past the heap buffer (CWE-680). The companion CVE-2026-55199 exposes a pre-auth denial-of-service path in the same function. Every libssh2 version through 1.11.1 is affected; no official patched release existed as of June 24. The embedded impact is amplified by how the library ships: statically linked into routers, NAS firmware, industrial controllers, edge gateways, and backup utilities — each of which requires an independent vendor firmware rebuild, signing cycle, and OTA deployment. This is not a single upstream update; it is a coordination problem across potentially millions of devices, most without auto-update mechanisms.

<b>2. Post: [Capability unlock] — 67KB model update now ships as a 475-byte patch</b>
Updating a machine learning model on a microcontroller used to mean retransmitting the entire .tflite file over the air every time you retrain.

TinyMLDelta changes the math. It computes a binary diff between the old and new model and ships only what changed. In a real test with a 67KB TFLite Micro sensor model, only 383 bytes differed between versions. The resulting OTA payload: 475 bytes — roughly a 140x reduction.

The runtime applies the patch atomically, validating firmware-to-model compatibility via TLV metadata before writing to flash. An incompatible patch is rejected cleanly before it touches the device, preventing partial-update corruption or silent accuracy regression.

This matters most for battery-powered field devices where bandwidth and flash wear are hard constraints — the kind of device where you cannot afford to retransmit 67KB every time the model improves.

Open source, TFLite Micro support today, Zephyr and Arduino OTA reference implementations on the roadmap.
