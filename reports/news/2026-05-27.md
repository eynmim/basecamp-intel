<b>═ 🔥 TOP STORY ═</b>
FreeRTOS 202604 LTS — released April 2026 and now the active LTS baseline — is the most consequential RTOS update of the quarter for every ESP32-S3 and STM32 developer running production firmware. FreeRTOS kernel v11.3.0 adds hardware MPU support expansion across Cortex-M and RISC-V targets, security hardening, and new hardware ports. coreMQTT jumps to v5.0.2 with full MQTT v5.0 protocol: topic aliases eliminate repeated string topic names on constrained over-the-air links, and the new request/response pattern makes interactive IoT workflows (OTA command/response, BLE-MQTT bridging) first-class protocol citizens rather than application-level hacks. coreSNTP v2.0.0 fixes the 2038 Unix timestamp overflow — devices deployed today will validate TLS certificates and timestamp sensor data correctly throughout their full operational lifetime without a future firmware emergency patch. The LTS tag guarantees security and critical bug fixes through April 2028, making 202604 the correct base for anything shipping mid-2026. Migration from FreeRTOS 202210 LTS is the recommended next maintenance step for any active project.

<b>═ 🔒 EMBEDDED CYBERSECURITY ═</b>
• <b>CVE-2026-41429 — Arduino ESP32 NBNS Stack Buffer Overflow, CVSS 8.8</b> [High] — A stack-based buffer overflow in the Arduino ESP32 NBNS (NetBIOS Name Service) handler allows any attacker on the local network to achieve unauthenticated RCE: the UDP port 137 parser trusts the attacker-controlled name_len field without bounds checking, corrupting the stack on any incoming NBNS request. Affects all variants (ESP32, ESP32-S2, ESP32-S3, ESP32-C3, ESP32-C6, ESP32-H2) running arduino-esp32 &lt; 3.3.8 — upgrade immediately; on AP-mode devices this is exploitable from any associated station. <a href="https://nvd.nist.gov/vuln/detail/CVE-2026-41429">Source →</a>
• <b>CVE-2026-31431 — Linux Kernel "Copy Fail" Root Privilege Escalation in CISA KEV</b> [High] — CISA added CVE-2026-31431 to the Known Exploited Vulnerabilities catalog: a CVSS 7.8 local privilege escalation in the Linux kernel via a copy_from_user failure-path race condition allowing any unprivileged user to obtain root. Embedded Linux gateways on ARM64/ARM32 targets (RPi CM4, i.MX8, Allwinner) running Ubuntu Core, Yocto, or Buildroot are in scope — apply kernel patches immediately on any multi-user or internet-facing gateway. <a href="https://thehackernews.com/2026/05/cisa-adds-actively-exploited-linux-root.html">Source →</a>
• <b>Arduino ESP32 CSRF OTA Firmware Endpoint — Unsigned Flash via Browser Visit</b> [Medium] — A second Arduino ESP32 advisory (GHSA-9vfw-wx65-c872) documents a CSRF vulnerability in the OTA firmware update endpoint exposed by the Arduino WebServer update handler, allowing a crafted HTTP request triggered from an attacker-controlled web page to flash unsigned firmware without authentication. Mitigation: restrict the OTA endpoint to localhost or add CSRF token validation; never expose the Arduino update handler on untrusted network interfaces. <a href="https://github.com/espressif/arduino-esp32/security/advisories/GHSA-9vfw-wx65-c872">Source →</a>
