<b>1. Post: [Capability unlock] — BT Core 6.2: a 20× latency drop that changes what BLE can do</b>
BLE just got 20× faster minimum connection intervals. That is not an incremental update.

Bluetooth Core 6.2 was released this week. The headline number: Shorter Connection Intervals (SCI) reduce the minimum BLE connection interval from 7.5 ms to 375 µs. A 20× reduction in the floor.

At 7.5 ms minimum, a BLE device exchanges data at most 133 times per second. At 375 µs, the ceiling reaches 2,666 Hz — comparable to USB HID polling rates. This opens a new class of BLE applications: real-time wireless HID, closed-loop haptic feedback under 1 ms round-trip, high-rate motion sensors that do not need proprietary radio stacks. Previously these required custom protocol layers or non-BLE hardware. Now they need a spec-compliant stack update.

The 6.2 release also adds Channel Sounding amplitude attack resilience — extending the secure BLE ranging system from 6.0 against a new relay attack vector.

Stack implementations (NimBLE, Zephyr BLE, nRF SoftDevice) need updates before SCI is available to application code. Watch the changelogs for your stack.

<b>3. Post: [Integration leap] — WireGuard VPN is now a native Zephyr RTOS networking primitive</b>
Embedded devices can now terminate authenticated encrypted VPN tunnels at the RTOS layer. No Linux. No separate daemon. No gateway.

Zephyr 4.4 (released April 2026) added WireGuard to its networking stack as a first-class protocol. <b>4. Post: [Physics-meets-architecture] — INT4 quantization on a microcontroller: the math works, the operator gap does not</b>
4-bit neural network weights on a Cortex-M4 are shipping in production firmware in 2026. The accuracy trade-off is manageable. The tooling gap is not.

TensorFlow Lite Micro now ships INT4 weight quantization. Moving from INT8 to INT4 halves model storage — a model needing 200 KB as INT8 fits in roughly 100 KB as INT4. On MCUs with 256 KB flash, that difference determines whether a model deploys at all.

The accuracy regression is real but bounded: expect 2–5% degradation on typical classification tasks with quantization-aware training. Acceptable for gesture, keyword spotting, and anomaly detection on most thresholds.

The problem is operator coverage. Not all TFLite Micro operators support INT4 yet. Some layers must stay INT8, creating mixed-precision deployments that require per-layer quantization configuration and complicate the build pipeline. Post-training quantization to INT4 without QAT produces larger drops.

The math is ready. The production tooling pipeline still has gaps to close. Worth benchmarking your specific model before committing INT4 to a release.
