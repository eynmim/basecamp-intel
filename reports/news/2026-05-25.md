<b>═ 🔥 TOP STORY ═</b>
DirtyDecrypt (CVE-2026-31635) PoC code was published May 18 by the Zellic/V12 team, exposing a copy-on-write guard missing in the Linux kernel's RxGK subsystem (rxgk_decrypt_skb()). When CONFIG_RXGK is compiled in, an unprivileged local user writes directly to a shared page-cache page without a private copy — escalating to root deterministically, no race condition required. Fedora, Arch, and openSUSE Tumbleweed ship with RxGK enabled and are fully exploitable; Debian Stable, Ubuntu LTS, and RHEL are not affected by default. For embedded Linux specifically: Yocto/Buildroot images typically disable CONFIG_RXGK, but any team using a rolling-release toolchain, a vendor BSP, or container-based field deployments must audit kernel config now — on an affected kernel a container escape is trivially chainable from this same flaw. The upstream patch was merged April 25; downstream distributors who have not synced are still exposed.
