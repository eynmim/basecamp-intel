<b>═ 🔥 TOP STORY ═</b>
Zephyr 4.4.0 shipped April 14, 2026 and its migration documentation has now stabilised — seven weeks post-release is the natural adoption window for production projects before the 4.5 cycle opens in October. The headline additions span every layer of the stack: WireGuard VPN is now kernel-native (no user-space library, no OpenVPN overhead), Wi-Fi Direct enables AP-free device-to-device connectivity, and the USB host-class driver framework expands significantly with UVC camera support when acting as a USB host. Three entirely new driver API families landed — OTP memory for permanent provisioning data, a Biometrics API, and a Wake-up Controller (WUC) API for fine-grained sub-µA sleep management. The release bundles Zephyr SDK 1.0, the first major SDK version with unified GNU toolchain, experimental Clang/LLVM, and pre-built multi-platform QEMU/OpenOCD. For any engineer currently running FreeRTOS or NimBLE-only stacks, Zephyr 4.4 is the release where the ecosystem breadth becomes operationally concrete: WireGuard alone eliminates an entire middleware dependency for secure tunnel firmware in gateway products.
