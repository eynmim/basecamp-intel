<b>═ 🔥 TOP STORY ═</b>
Dream Security disclosed CVE-2026-32746 on May 14, 2026 — a stack buffer overflow in GNU Inetutils telnetd (CVSS 9.8 Critical) that grants an unauthenticated remote attacker root-level RCE on any host running the affected daemon. The flaw sits in the telnet option-processing path and requires zero credentials, meaning any network-reachable telnet port is fully exploitable with a single crafted packet sequence. Dream Security explicitly called out ICS and OT environments, where legacy embedded Linux gateways and panel PCs routinely ship telnetd enabled for "commissioning convenience" and leave the port open post-deployment. Embedded engineers must run <i>grep -r inetutils</i> across their Yocto/Buildroot recipes, disable telnetd immediately on all connected targets, and replace it with SSH — this is a one-line fix that eliminates a critical attack surface that has existed unexamined in firmware images for years.
