<b>═ 🔥 TOP STORY ═</b>
"Dirty Frag" — a two-CVE Linux kernel privilege escalation chain (CVE-2026-43284 + CVE-2026-43500) — was disclosed May 8 by researcher Hyunwoo Kim after a coordinated embargo broke early, and a working public PoC already exists that elevates any unprivileged local user to root in a single command. CVE-2026-43284 exploits how the IPsec ESP receive path decrypts over paged buffers not privately owned by the kernel; when combined with splice(2)/sendfile(2), an unprivileged process retains write primitives into the page cache. CVE-2026-43500 extends the same primitive via the RxRPC subsystem — and as of May 20 its upstream kernel patch is still not published in NVD, leaving all unmitigated systems open. For embedded engineers: any Linux-based IoT gateway, edge server, or dev machine running an unpatched kernel is at full root exposure from any local attacker foothold — container escapes, web-shell pivot, or low-priv SSH session all qualify. Patch CVE-2026-43284 immediately via distro kernels; monitor NVD for CVE-2026-43500's patch before it lands silently in your next kernel update.
